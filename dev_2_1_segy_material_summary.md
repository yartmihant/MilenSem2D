## Методы обработки

### 1. Анализ формата SEG-Y

**Используемая библиотека:** `segyio` (Python)

**Структура SEG-Y файла:**
- **Текстовый заголовок:** 3200 байт ASCII текста (опционально)
- **Бинарный заголовок:** 400 байт метаданных
- **Трассы:** Набор сейсмических трасс, каждая содержит:
  - Заголовок трассы: 240 байт
  - Данные трассы: N сэмплов × 4 байта (IEEE float)

**Ключевые поля бинарного заголовка:**
- `JobID`: идентификатор работы
- `Samples`: количество сэмплов в трассе
- `Interval`: интервал между сэмплами (микросекунды)
- `Format`: формат данных (5 = IEEE float)

**Ключевые поля заголовка трассы:**
- `TRACE_SEQUENCE_FILE`: номер трассы в файле
- `INLINE_3D`: номер inline (для 3D)
- `CROSSLINE_3D`: номер crossline (для 3D)
- `CDP_X`: X-координата CDP (общей глубинной точки)
- `CDP_Y`: Y-координата CDP
- `TRACE_SAMPLE_COUNT`: количество сэмплов в трассе
- `TRACE_SAMPLE_INTERVAL`: интервал выборки (микросекунды)

**Особенности формата Tesseral:**
- Формат данных: IEEE float (4 байта)
- Интервал: 2000 мкс (в примере)
- Координаты CDP: целочисленные значения в единицах карты

### 2. Преобразование данных материала

**Обратное преобразование:** (E, ν, ρ) → (Vp, Vs, ρ)

**Формулы:**

Скорость продольной волны:
```
Vp = sqrt(E/ρ × (1-ν)/((1+ν)(1-2ν)))
```

Скорость поперечной волны:
```
Vs = sqrt(E/(2ρ(1+ν)))
```

Плотность в г/см³:
```
density = ρ / 1000.0
```

где:
- E - модуль Юнга (Па)
- ν - коэффициент Пуассона (безразмерный)
- ρ - плотность (кг/м³)

**Диапазоны преобразованных данных:**
- Vp: 1,209.4 - 5,267.8 м/с
- Vs: 664.3 - 2,896.5 м/с
- Плотность: 1.814 - 2.578 г/см³

### 3. Запись в формат SEG-Y

**Функция:** `write_segy_2d(filename, data, dx, dy, x_origin, y_origin)`

**Алгоритм:**
1. Создание спецификации SEG-Y:
   ```python
   spec = segyio.spec()
   spec.format = 5  # IEEE float
   spec.samples = range(ny)  # индексы сэмплов
   spec.tracecount = nx  # количество трасс
   spec.interval = int(dy * 1000)  # интервал в мкс
   ```

2. Запись бинарного заголовка:
   ```python
   f.bin = {
       segyio.BinField.JobID: 1,
       segyio.BinField.Samples: ny,
       segyio.BinField.Interval: spec.interval,
       segyio.BinField.Format: 5
   }
   ```

3. Запись трасс (цикл по X):
   ```python
   for i in range(nx):
       # Заголовок трассы
       f.header[i] = {
           TRACE_SEQUENCE_FILE: i + 1,
           INLINE_3D: i + 1,
           CROSSLINE_3D: 1,
           CDP_X: int(x_origin + i * dx),
           CDP_Y: int(y_origin),
           TRACE_SAMPLE_COUNT: ny,
           TRACE_SAMPLE_INTERVAL: spec.interval
       }
       # Данные трассы (столбец по Y)
       f.trace[i] = data[i, :].astype(np.float32)
   ```

**Параметры записи:**
- dx = 5.0 м (шаг по X)
- dy = 5.0 м (шаг по Y)
- Интервал = 5000 мкс (5 м × 1000)
- Формат: IEEE float (32-bit)

**Интерпретация координат:**
- Каждая трасса = вертикальный профиль (столбец по Y)
- Номер трассы = позиция по X
- Номер сэмпла = позиция по Y (глубина)

### 4. Верификация данных

**Проверка записанных файлов:**
1. Повторное открытие файлов через `segyio`
2. Чтение всех трасс: `segyio.tools.collect(f.trace[:])`
3. Проверка формы массива: должна совпадать с исходной
4. Проверка диапазона значений: должен совпадать с исходным
5. Проверка метаданных: количество трасс и сэмплов

**Критерии успешности:**
- Форма данных: (2350, 550) ✓
- Диапазоны сохранены ✓
- Нет потери точности (IEEE float) ✓

## Технические детали

### Формат SEG-Y

**Стандарт:** SEG-Y Revision 1 (Society of Exploration Geophysicists)

**Структура файла:**
```
[Текстовый заголовок]     3200 байт (опционально)
[Бинарный заголовок]      400 байт
[Трасса 1 - Заголовок]    240 байт
[Трасса 1 - Данные]       N × 4 байт (IEEE float)
[Трасса 2 - Заголовок]    240 байт
[Трасса 2 - Данные]       N × 4 байт
...
[Трасса M - Заголовок]    240 байт
[Трасса M - Данные]       N × 4 байт
```

**Размер файла:**
```
Размер = 3200 + 400 + M × (240 + N × 4) байт
       = 3600 + 2350 × (240 + 550 × 4)
       = 3600 + 2350 × 2440
       = 5,737,600 байт
       ≈ 5.5 MB
```

где M = 2350 трасс, N = 550 сэмплов.

### Производительность

**Время выполнения операций:**
- Загрузка примера Tesseral (60,501 × 1,501): ~2 сек
- Загрузка декартовых данных (2350 × 550): ~0.5 сек
- Преобразование (E, ν, ρ) → (Vp, Vs, ρ): ~0.3 сек
- Запись одного SEG-Y файла: ~1.5 сек
- Запись трех файлов: ~4.5 сек
- Верификация всех файлов: ~2 сек
- **Общее время:** ~10 секунд

**Потребление памяти:**
- Загрузка данных: ~50 MB
- Преобразование: ~100 MB
- Запись файлов: ~20 MB на файл
- **Пиковое потребление:** ~150 MB

**Размер данных:**
- Входные данные (npz): ~30 MB (сжатые)
- Выходные данные (3 × SEG-Y): 3 × 5.5 MB = 16.5 MB
- **Общий объем:** ~46.5 MB

### Точность данных

**Формат хранения:** IEEE 754 single precision (32-bit float)

**Характеристики:**
- Мантисса: 23 бита (~7 значащих цифр)
- Экспонента: 8 бит (диапазон: 10⁻³⁸ до 10³⁸)
- Относительная точность: ~10⁻⁷

**Проверка точности:**

rho_segy = load_segy_model('data/dev_2_1_Density_model.sgy') * 1000

# Обратное преобразование
nu_segy = (vp_segy**2 - 2*vs_segy**2) / (2*(vp_segy**2 - vs_segy**2))
E_segy = 2 * rho_segy * vs_segy**2 * (1 + nu_segy)

# Загрузка исходных данных
data_orig = np.load('data/dev_1_7_material_grids.npz')
E_orig = data_orig['material_grid'][:, :, 0]
nu_orig = data_orig['material_grid'][:, :, 1]
rho_orig = data_orig['material_grid'][:, :, 2]

# Сравнение
rel_error_E = np.abs(E_segy - E_orig) / E_orig
rel_error_nu = np.abs(nu_segy - nu_orig) / nu_orig
rel_error_rho = np.abs(rho_segy - rho_orig) / rho_orig

print(f"Относительная ошибка E: {rel_error_E.max():.2e}")
print(f"Относительная ошибка ν: {rel_error_nu.max():.2e}")
print(f"Относительная ошибка ρ: {rel_error_rho.max():.2e}")
