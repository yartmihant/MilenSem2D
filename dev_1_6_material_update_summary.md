# Глава VI: Генерация материала под новую структуру слоев

## Краткий реферат

В данной главе выполнена генерация упругих свойств материала для новой 75-слойной структуры, созданной в предыдущей главе. Реализована интерполяция сейсмических параметров (Vp, Vs, плотность) между скважинами с использованием лагранжевых координат внутри каждого слоя. Создана эйлерова сетка материала с адаптивным разрешением, зависящим от толщины слоев. Материал подготовлен для использования в конечноэлементном моделировании в Fidesys.

## Задачи главы

### Задача 1: Подготовка данных послойно
Загрузка геометрии слоев и каротажных данных, разделение данных по 75 слоям.

### Задача 2: Расчет количества элементов по слоям
Определение оптимального количества элементов для каждого слоя на основе его толщины.

### Задача 3: Интерполяция в лагранжевой сетке
Создание лагранжевой параметризации внутри каждого слоя и интерполяция свойств методом кригинга.

### Задача 4: Построение эйлеровой сетки
Конвертация лагранжевых координат в эйлеровы (реальное пространство) с созданием структурированной сетки.

### Задача 5: Разделение данных на слои
Подготовка материалов для каждого слоя отдельно для записи в FC модель.

### Задача 6: Генерация FC модели с материалами
Создание FC файлов с полными материалами для всех слоев, включая демпфирование на краях модели.

## Методы обработки

### 1. Парсинг каротажных данных

**Функция:** `parse_well_data(filepath)`

**Алгоритм:**
1. Чтение текстового файла, пропуск заголовка
2. Извлечение значений Vp, плотности, Vs
3. Создание правильных глубин с шагом 10 м, начиная с 10 м
4. Формирование массивов параметров

**Результат:** Массивы глубин и параметров (Vp, rhob, Vs) для каждой скважины.

### 2. Подготовка послойных данных

**Алгоритм:**
1. Добавление нулевой поверхности (глубина 0) к границам слоев
2. Для каждого слоя определение границ в обеих скважинах
3. Извлечение каротажных данных в пределах каждого слоя
4. Создание структуры данных для 75 слоев

**Структура данных слоя:**
```python
{
    'formation': str,                  # Название формации
    'well1_top_depth': float,          # Верх слоя в скважине 1
    'well1_bottom_depth': float,       # Низ слоя в скважине 1
    'well2_top_depth': float,          # Верх слоя в скважине 2
    'well2_bottom_depth': float,       # Низ слоя в скважине 2
    'well1_data': array[N1, 3],        # Данные скважины 1 [Vp, rhob, Vs]
    'well2_data': array[N2, 3],        # Данные скважины 2 [Vp, rhob, Vs]
    'well1_depths': array[N1],         # Глубины замеров в скважине 1
    'well2_depths': array[N2]          # Глубины замеров в скважине 2
}
```

### 3. Расчет количества элементов

**Параметры:**
- Максимальная толщина элемента: 10 м
- Минимальное количество элементов: 1

**Алгоритм:**
```python
layer_thickness = mean(layer_bottom - layer_top)
n_elements = max(1, ceil(layer_thickness / 10.0))
```

**Результат:** Массив `layer_elements_count[75]` с количеством элементов для каждого слоя.

### 4. Интерполяция в лагранжевой сетке

**Функция:** `interpolate_layer_properties(layer_data, n_eta_nodes)`

**Лагранжевы координаты:**
- ξ (xi): горизонтальная координата вдоль профиля, соответствует distances
- η (eta): вертикальная координата внутри слоя, η ∈ [0, 1]
  - η = 0 соответствует верхней границе слоя
  - η = 1 соответствует нижней границе слоя

**Алгоритм:**
1. Создание сетки в лагранжевых координатах [1176 × n_eta_nodes × 3]
2. Для каждого узла η:
   - Определение соответствующей глубины в каждой скважине
   - Извлечение значений параметров в скважинах
   - Интерполяция между скважинами методом Ordinary Kriging
3. Обработка случаев с малым разбросом данных (std < 1e-9)

**Метод интерполяции:**
- **Основной:** Ordinary Kriging (PyKrige) с линейным вариограмом
- **Запасной:** Линейная интерполяция (при сбое кригинга)
- **Константа:** При отсутствии вариации данных

**Результат:** Массив свойств материала в лагранжевых координатах для каждого слоя.

### 5. Построение эйлеровой сетки

**Система координат:**
- Эйлеровы координаты (x, y): реальное физическое пространство
- x: расстояние вдоль профиля (0 - 11750 м)
- y: глубина (0 - 2750 м)

**Параметры сетки:**
- `n_xi_nodes = 1176` - количество узлов по горизонтали
- `n_z_nodes` - количество узлов по вертикали (сумма элементов + 1)
- Переменное разрешение по вертикали (зависит от толщины слоев)

**Алгоритм:**
1. Для каждой горизонтальной позиции x:
   - Извлечение границ всех слоев в этой точке
   - Для каждого слоя:
     - Создание n_elements + 1 узлов между границами
     - Интерполяция свойств из лагранжевой сетки
     - Запись координат и свойств в эйлерову сетку

**Структура результата:**
- `eulerian_coords[1176, n_z_nodes, 2]` - координаты узлов (x, y)
- `eulerian_props_nodes[1176, n_z_nodes, 3]` - свойства (Vp, rhob, Vs)

### 6. Конвертация параметров

**Преобразование:** (Vp, Vs, ρ) → (E, ν, ρ)

**Функции:**
- `wave_speeds(E, nu, rho)` - прямое преобразование
- `inverse_wave_speeds(Vp, Vs, rho)` - обратное преобразование

**Формулы:**

Коэффициент Пуассона:
```
ν = (Vp² - 2Vs²) / (2(Vp² - Vs²))
```

Модуль Юнга:
```
E = 2ρVs²(1 + ν)
```

**Результат:** Свойства материала в формате [E (Па), ν (безразмерный), ρ (кг/м³)]

### 7. Разделение на слои

**Алгоритм:**
1. Для каждого слоя определение индексов узлов по вертикали
2. Извлечение координат и свойств для всех горизонтальных позиций
3. Добавление узлов из соседних слоев (overlap) для гладкости
4. Формирование структуры данных слоя

**Структура данных:**
```python
{
    'formations': str,               # Название формации
    'coords': array[N, 2],          # Координаты (x, y)
    'properties': array[N, 3]       # Свойства (E, ν, ρ)
}
```

### 8. Генерация JOU-скриптов

**Файл:** `data/dev_1_6_model_material_stub.jou`

**Содержимое:**
- Удаление существующих таблиц, блоков, материалов
- Создание блоков для каждого слоя (3 поверхности на блок)
- Создание таблиц (4 на материал):
  - Table 4i: MODULUS (E)
  - Table 4i-1: POISSON (ν)
  - Table 4i-2: DENSITY (ρ)
  - Table 4i-3: MASS_MATRIX_DAMPING
- Заполнение первых 2 строк данных (заглушка)
- Присвоение материалов блокам (order 4)

**Примечание:** Полные данные записываются позже через FC модель для ускорения загрузки.

### 9. Создание FC модели

**Функция:** Прямое редактирование структуры FCModel

**Алгоритм:**
1. Загрузка базовой FC модели с заглушками
2. Для каждого слоя (материала):
   - Обновление координат и значений E (MODULUS)
   - Обновление координат и значений ν (POISSON)
   - Обновление координат и значений ρ (DENSITY)
   - Расчет и запись MASS_MATRIX_DAMPING

**Демпфирование на краях модели:**

Условия применения:
- x < 250 м (левый край, 250 м демпфирующий слой)
- x > 11500 м (правый край, 250 м демпфирующий слой)
- y > 2450 м (нижний край, 300 м демпфирующий слой)

Формулы:
```python
if x < 250:
    damping_x = 100 * ((250 - x) / 250)²
elif x > 11500:
    damping_x = 100 * ((x - 11500) / 250)²

if y > 2450:
    damping_y = 100 * ((y - 2450) / 350)²

mass_matrix_damping = max(damping_x, damping_y)
```

**Параметры:**
- Максимальное демпфирование: 100
- Квадратичный закон нарастания
- Применяется независимо по осям

### 10. Генерация упрощенного материала

### Распределение элементов по слоям

**Примеры (толщина → количество элементов):**

**Статистика:**
- Минимум: 1 элемент на слой

**Коэффициент Пуассона (ν):**
- Минимум: ~0.10
- Максимум: ~0.48
- Типичный диапазон: 0.25-0.35

**Плотность (ρ):**
- Минимум: ~1,900 кг/м³ (рыхлые породы)

## Технические детали

### Лагранжевы координаты

**Преимущества:**
- Следуют геологической истории слоев
- Обеспечивают гладкость внутри слоя
- Упрощают интерполяцию между скважинами

**Параметризация:**
- η ∈ [0, 1] для каждого слоя независимо
- Линейное отображение глубины на η
- Сохранение стратиграфической структуры

### Метод Ordinary Kriging

**Параметры:**
- Модель вариограммы: linear
- Verbose: False (без вывода)
- Enable plotting: False (без графиков)

**Обработка ошибок:**
- При сбое кригинга: fallback на линейную интерполяцию
- При std < 1e-9: константное значение
- При отсутствии данных: пропуск слоя

### Производительность

**Время выполнения:**
- Парсинг данных: ~1 сек
- Интерполяция всех слоев: ~30-40 сек
- Построение эйлеровой сетки: ~5-10 сек
- Создание FC модели: ~10-20 сек
- **Общее время:** ~1 минута

**Потребление памяти:**
- Загрузка данных: ~50 MB
- Интерполяция: ~200-300 MB
- Эйлерова сетка: ~100 MB
- FC модель: ~1-2 GB (в памяти)
- **Пиковое потребление:** ~2-3 GB
