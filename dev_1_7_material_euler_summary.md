# Глава VII: Эйлерова декартова сетка материала

## Краткий реферат

В данной главе реализован альтернативный подход к представлению материала: поверх криволинейной слоистой геометрии натянута абсолютно ровная декартова сетка материала. Такой подход обеспечивает совместимость с методом конечных разностей, используемым коллегами-геофизиками, и позволяет отладить методы интерполяции материалов. Создана декартова сетка дискретизации 5×5 м для передачи геофизикам и угрубленная версия 25×5 м для записи в FC модель. Выполнен расчет адаптивных размеров элементов сетки на основе минимальных скоростей продольных волн.

## Научная задача

В предыдущей главе был создан материал, следующий криволинейной геометрии слоев в лагранжевых координатах. Однако такой подход имеет ограничения:

1. **Несовместимость с методом конечных разностей:** Коллеги-геофизики используют прямоугольные сетки
2. **Сложность обмена данными:** Криволинейные координаты затрудняют передачу материалов
3. **Ограниченная применимость:** Не все коды поддерживают материалы в лагранжевых координатах

**Решение:** Декартова эйлерова сетка материала

**Преимущества:**
- Совместимость с методом конечных разностей
- Простота обмена данными
- Универсальность представления
- Возможность отладки методов интерполяции

**Особенность:** В МСЭ такой подход используется редко, но он возможен и полезен для междисциплинарного сотрудничества.

## Методы обработки

### 1. Определение принадлежности к слоям

**Функция:** `find_layer_at_point(x, y, distances, all_boundaries)`

**Алгоритм:**
1. Находим ближайший профиль по координате x
2. Извлекаем границы всех слоев в этой точке
3. Определяем, между какими границами находится y
4. Возвращаем номер слоя (0-74) или -1 если вне модели

**Применение:** Создание массива `layer_indexes_grid[Nx, Ny]` с индексами слоев для каждой точки декартовой сетки.

### 2. Построение декартовой сетки

**Параметры:**
- dx = 5 м (шаг по горизонтали)
- dy = 5 м (шаг по вертикали)

**Координаты узлов:**
```python
x_cells = arange(x_min + dx/2, x_max, dx)  # центры ячеек
y_cells = arange(y_min + dy/2, y_max, dy)  # центры ячеек
X_grid, Y_grid = meshgrid(x_cells, y_cells, indexing='ij')
```

**Размеры:**
- Nx = 2,350 (узлов по X)
- Ny = 550 (узлов по Y)
- Всего узлов: 1,292,500

**Структура данных:**
- `coords_grid[Nx, Ny, 2]` - координаты (x, y) каждого узла
- `layer_indexes_grid[Nx, Ny]` - индекс слоя для каждого узла
- `material_grid[Nx, Ny, 3]` - свойства материала (E, ν, ρ)

### 3. Построение интерполяторов

**Метод:** CloughTocher2DInterpolator (scipy)

**Характеристики:**
- Кусочно-кубическая интерполяция
- C1-непрерывность (гладкие производные)
- Робастность к нерегулярным сеткам
- Fill value: среднее значение свойства (для экстраполяции)

**Алгоритм:**
1. Для каждого слоя (i = 0..74):
   - Извлекаем координаты узлов материала из лагранжевой сетки
   - Извлекаем значения свойств (E, ν, ρ)
   - Строим три интерполятора (по одному для каждого свойства):
     - `interp_E`: интерполятор модуля Юнга
     - `interp_nu`: интерполятор коэффициента Пуассона
     - `interp_rho`: интерполятор плотности

**Результат:** Список из 75 структур интерполяторов:
```python
{
    'formation': str,
    'n_points': int,
    'interp_E': CloughTocher2DInterpolator,
    'interp_nu': CloughTocher2DInterpolator,
    'interp_rho': CloughTocher2DInterpolator
}
```

### 4. Интерполяция на декартовую сетку

**Алгоритм:**
1. Для каждого слоя (i = 0..74):
   - Находим все узлы декартовой сетки, принадлежащие слою
   - Извлекаем координаты этих узлов
   - Применяем интерполяторы пакетно ко всем узлам:
     ```python
     E_values = interp_E(layer_coords)
     nu_values = interp_nu(layer_coords)
     rho_values = interp_rho(layer_coords)
     ```
   - Записываем интерполированные значения в `material_grid`

**Оптимизация:** Пакетная интерполяция всех узлов слоя одновременно (векторизация).

**Результат:** Заполненный массив `material_grid[Nx, Ny, 3]` со свойствами материала.

### 5. Расчет скоростей волн

**Формулы:**

Скорость продольной волны:
```
Vp = sqrt(E/ρ × (1-ν)/((1+ν)(1-2ν)))
```

Скорость поперечной волны:
```
Vs = sqrt(E/(2ρ(1+ν)))
```

**Результат:** Массивы `Vp[Nx, Ny]` и `Vs[Nx, Ny]` в м/с.

### 7. Расчет адаптивной сетки FDS

**Критерий:** Размер элемента должен составлять 1/4 длины продольной волны минимальной скорости.

**Параметры:**
- Частота: f = 30 Hz
- Элементов на длину волны: el_in_wave = 4
- Длина волны: λ = Vp / f

**Максимальный размер элемента:**
```python
max_len_per_depth = min(Vp, axis=0) / 30 / 4
```

где `min(Vp, axis=0)` - минимальная скорость Vp на каждой глубине.

**Расчет по слоям:**
```python
max_param_per_layer[i] = max(max_len_per_depth[0:layer_depth[i]])
```

**Коррекция первого слоя:** Устанавливаем строго 10 м для удобства расстановки ресиверов.

**Результат:** Массив `max_param_per_layer[75]` с максимальными размерами элементов для каждого слоя.

**Обоснование:** По таблице Ухнова для 2D МСЭ 5-го порядка достаточно 2 элементов на длину волны для достижения точности 1%. У нас 4 элемента, что обеспечивает хороший запас точности.

**Примечание:** Таблица Уханова составлялась для пульса Берлаге в однородном случае, поэтому является оценкой снизу (минимум, который нельзя нарушать).

### 8. Расчет интервалов сетки

**Условие сетки pave:** Четная сумма интервалов на границах.

**Обеспечение:** Равная четность в горизонтальных и вертикальных группах.

**Горизонтальные группы:**
- Группа 1 (левая часть): длина 4,250 м
- Группа 2 (центральная часть): длина 3,250 м
- Группа 3 (правая часть): длина 4,250 м
- Все горизонтальные группы: нечетное число элементов

**Расчет интервалов:**
```python
h1_interval = round(425 / max_param_per_layer) * 10 + 5
h2_interval = round(325 / max_param_per_layer) * 10 + 5
h3_interval = round(425 / max_param_per_layer) * 10 + 5
```

**Вертикальные группы:**
- Высота слоя в скважинах: `well1_layer_height`, `well2_layer_height`
- Интервалы: `ceil(layer_height / max_param_per_layer)`
- Коррекция четности для совместимости между скважинами

**Алгоритм коррекции четности:**
```python
for i in range(n_layers):
    parity1 = well1_layer_interval[i] % 2
    parity2 = well2_layer_interval[i] % 2
    if parity1 != parity2:
        if well1_layer_interval[i] > well2_layer_interval[i]:
            well1_layer_interval[i] += 1
        else:
            well2_layer_interval[i] += 1
```

**Базовая модель:** `data/dev_1_7_model_material_stub.fc`

- Угрубление в 5 раз по X существенно уменьшает размер модели
- Разрешение по Y остается прежним (5 м)

**Алгоритм:**
1. Загрузка базовой FC модели с заглушками
2. Для каждого слоя (материала):
   - Извлечение узлов, принадлежащих слою
   - Обновление таблиц координат и значений:
     - MODULUS (E)
     - POISSON (ν)
     - DENSITY (ρ)
     - MASS_MATRIX_DAMPING

**Демпфирование на краях:**

Условия:
- x < 250 м (левый край)
- x > 11,500 м (правый край)
- y > 2,500 м (нижний край)

Формулы:
```python
if x < 250:
    damping_x = 100 * ((250 - x) / 250)²
elif x > 11500:
    damping_x = 100 * ((x - 11500) / 250)²

if y > 2500:
    damping_y = 100 * ((y - 2500) / 250)²

mass_matrix_damping = max(damping_x, damping_y)
```

## Сравнение подходов

### Лагранжев материал (Глава VI)
**Преимущества:**
- Следует геологической структуре
- Гладкость внутри слоев
- Естественная параметризация

**Недостатки:**
- Сложность обмена данными
- Несовместимость с конечными разностями
- Требует специальной обработки

### Эйлеров декартов материал (Глава VII)
**Преимущества:**
- Совместимость с конечными разностями
- Простота обмена данными
- Универсальность представления
- Легкая визуализация

**Недостатки:**
- Не следует геологической структуре
- Может нарушать гладкость на границах слоев
- Больший объем данных

**Применение:**
- Лагранжев: для высокоточных расчетов МСЭ
- Эйлеров: для обмена с геофизиками (конечные разности)
- Оба подхода дополняют друг друга

## Использование результатов

### Загрузка декартовой сетки
```python
import numpy as np

# Загрузка данных
data = np.load('data/dev_1_7_material_grids.npz')
material_grid = data['material_grid']      # [2350, 550, 3]
coords_grid = data['coords_grid']          # [2350, 550, 2]
layer_indexes_grid = data['layer_indexes_grid']  # [2350, 550]

# Извлечение свойств
E_grid = material_grid[:, :, 0]   # Модуль Юнга
nu_grid = material_grid[:, :, 1]  # Коэффициент Пуассона
rho_grid = material_grid[:, :, 2] # Плотность

# Расчет скоростей
Vp = np.sqrt(E_grid / rho_grid * (1 - nu_grid) / 
             ((1 + nu_grid) * (1 - 2*nu_grid)))
Vs = np.sqrt(E_grid / (2 * rho_grid * (1 + nu_grid)))

print(f"Размер сетки: {material_grid.shape}")
print(f"Диапазон Vp: {Vp.min():.0f} - {Vp.max():.0f} м/с")
```

## Технические детали

### Интерполятор Clough-Tocher 2D

**Характеристики:**
- Кусочно-кубическая интерполяция на триангуляции Делоне
- C1-непрерывность (непрерывны первые производные)
- Хорошая обработка нерегулярных данных
- Минимальная кривизна поверхности

**Преимущества над линейной интерполяцией:**
- Гладкость полей материала
- Отсутствие разрывов производных
- Лучшая точность в МСЭ высокого порядка

**Преимущества над RBF:**
- Быстрее для больших массивов данных
- Меньше потребление памяти
- Более предсказуемое поведение

### Производительность

**Время выполнения основных этапов:**
- Загрузка данных: ~2 сек
- Построение интерполяторов (75 слоев): ~15-20 сек
- Интерполяция на декартовую сетку: ~40-60 сек
- Визуализация (6 графиков): ~10-15 сек
- Расчет адаптивной сетки: ~5 сек
- Создание FC модели: ~20-30 сек
- **Общее время:** ~2-3 минуты

**Потребление памяти:**
- Загрузка данных: ~100 MB
- Интерполяторы: ~200 MB
- Декартовая сетка: ~300 MB
- FC модель: ~500 MB
- **Пиковое потребление:** ~800 MB - 1 GB

**Оптимизации:**
- Пакетная интерполяция узлов каждого слоя
- Векторизация операций NumPy
- Сжатие выходных файлов (npz)

### Контроль качества

**Проверки:**
1. Непрерывность полей на границах слоев
2. Отсутствие нефизических значений (E > 0, 0 < ν < 0.5, ρ > 0)
3. Соответствие скоростей волн исходным каротажным данным
4. Гладкость полей (визуальная проверка)

**Диагностика:**
- Минимальные/максимальные значения всех свойств
- Гистограммы распределений
- Профили вдоль скважин

## Выводы

Успешно реализован альтернативный подход к представлению материала на декартовой сетке:

1. **Декартова сетка 5×5 м:** Создана равномерная сетка материала, совместимая с методом конечных разностей и удобная для обмена данными с коллегами-геофизиками

2. **Качественная интерполяция:** Использован метод Clough-Tocher 2D, обеспечивающий C1-непрерывность и гладкость полей материала

3. **Угрубленная версия для FC:** Создана экономичная версия материала (25×5 м) для записи в FC модель, уменьшающая размер файла в ~5 раз при сохранении точности

4. **Адаптивная сетка FDS:** Разработана методика расчета оптимальных размеров элементов на основе минимальных скоростей продольных волн и частоты сигнала

5. **Полная автоматизация:** Созданы JOU-скрипты для автоматического построения адаптивной сетки в Fidesys с учетом особенностей каждого слоя

6. **Всесторонняя визуализация:** Подготовлены графики всех важных полей (E, ν, ρ, Vp, Vs, индексы слоев) для контроля качества

7. **Демпфирование на краях:** Реализовано квадратичное нарастание демпфирования на границах модели для поглощения отраженных волн

8. **Междисциплинарное сотрудничество:** Обеспечена возможность передачи материалов коллегам, работающим с методом конечных разностей

9. **Дополнительность подходов:** Декартов материал дополняет лагранжев материал из Главы VI, предоставляя альтернативное представление для различных задач

Созданные материалы готовы для:
- Численного моделирования методом спектральных элементов
- Передачи коллегам-геофизикам (метод конечных разностей)
- Валидации и сравнения различных подходов
